AIによるレポート


このAIによるレポートを人格データとしてSNSのユーザーの性質情報に書き込みタイムラインを各種形成します。

反応するかどうかまたどんな反応をするかをSNSの指標を使い定型化します。

### 反応対応表
|反応 |必要条件 |
|-----|----------|
|無視 |0 |
|いいね |好悪度+20 |
|リツイート |有効性+30  好悪度+30 |
|コメント |好悪度+50 or 好悪度-10 |


### パラメーター表
|名前 |説明 |
|---------|--------------|
|好感度 |好感度と険悪度を示す指標 -であるほど険悪 +であるほど好き |
|憎悪度 |本人の人格に従って憎悪、受けいることが困難な場合に出力をするシステム |
|好悪度 |好感度 - 憎悪度での総合的な好感度 |
|有効性 |この投稿が自身のいるコミュニティーで役立つか、他の人にも伝えたいかなどの伝えたい欲求のことを指す |


好感度の上下をどのように感知するか?  

AIに対して    

入力
```inputData.json
{
    "like":42,
    "bad":39,
    "inputContext":"あほ",
}
```

出力
```
{
    "like":30,
    "bad":46,
    "effective":2
    "outputContext":"無視"
}
```
などのようにJSON形式での出力を促し、それ以外の発言をカットし、
それをもとにAIに対して学習を行うというような仕組み。


## サーバー
こちらは分散型SNSの仕組みを採用しており
各自ユーザーがサーバーを所有、管理し、ベースのシステムとしてこのシステムを共有しています。
ユーザーはデジタルツインの適当なサーバーに参加するか、自身でサーバーを所有するか、してそれぞれの
サーバーで独自のデジタルツインを実装します。
